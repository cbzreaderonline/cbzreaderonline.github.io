Before you write in

Many common questions are already answered on the FAQ section of the homepage, covering file formats, supported platforms, and offline use. It might save you a wait.

What to include in a bug report

  • The browser and device you were using, for example Chrome on Windows or Safari on an iPhone.
  • Whether the file was a CBZ or a CBR archive.
  • What happened, for example the file would not open, a page would not display, or the theme toggle was not working.

This helps us reproduce the issue quickly and fix it faster.
